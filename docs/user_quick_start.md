# 普通用户极简使用说明

这是一款给 Excel 本地化流程用的桌面工具，主要用于主表、小表、术语和批处理任务。

## 启动方式

- 如果你拿到的是打包好的 Windows 版本，直接打开 `TM_builder.exe`。
- 如果你拿到的是源码环境，请让维护者先完成安装，再运行：

```powershell
.\scripts\python.cmd app.py
```

## 怎么选功能

- 想把主表内容批量写进小表，用 `Content Sync -> Master->Target`。
- 想把小表里的翻译反向写回主表，用 `Content Sync -> Target->Master`。
- 想一次跑很多任务，用 `Content Sync -> Batch`。
- 想处理整批 Excel 的辅助操作，用 `Utilities` 里的工具页签。
- 想更新、合并或补充主表，用 `Master Update` 里的工具页签。

## 功能总览

### Content Sync

- `Master->Target`
  - 用主表批量更新小表文件夹里的 Excel。
  - 适合把主表中的译文或其他内容列同步到多个小表。
  - 可设置列号、更新列数、仅填空、允许空白写入、兼容性后处理。

- `Target->Master`
  - 把小表里的译文反向回写到主表。
  - 适合翻译已经分散在多个小表里，需要统一收回主表时使用。
  - 可设置列号、仅填空、允许空白写入。

- `Batch`
  - 批量执行多组同步任务。
  - 支持两种批处理模式：
    - `master_to_target_single`：批量执行“主表 -> 小表”。
    - `target_to_master_reverse`：批量执行“小表 -> 主表”。
  - 可添加多条 Job，也可导入、保存、导出 JSON 配置模板。
  - 支持 `Precheck` 预检查、`Run Batch` 正式执行、`Continue on error` 出错后继续。
  - 支持通过自动填充配置，快速补全批处理 Job。

### Utilities

- `Column Clear`
  - 对目标文件夹里的所有 Excel 批量处理某一列。
  - 提供三个动作：
    - `清空列`：清空指定列内容。
    - `插入列`：在指定位置插入新列。
    - `删除列`：删除指定列，删除不可撤销。

- `Compatibility`
  - 用 Excel 逐个打开并保存目标文件夹里的 Excel。
  - 适合处理兼容性保存、格式修复、某些文件需要重新另存后才能继续处理的情况。
  - 依赖本机 Microsoft Excel 和 `pywin32`。

- `Deep Replace`
  - 用源文件夹里“同名”的 Excel，替换目标文件夹中的文件。
  - 适合把一批新版文件直接覆盖到旧目录里。
  - 如果替换失败，程序会尝试回滚到备份。

- `Untranslated Stats`
  - 统计一批小表里的未翻译内容，并导出 Excel 报表。
  - 支持两种统计模式：
    - `中文字符模式`
    - `英文词模式`
  - 可设置原文列、译文列，输出文件也可以自定义。

- `Term Extractor`
  - 从输入文件夹里提取术语，并导出结果 Excel。
  - 需要选择输入文件夹、规则 JSON、输出文件。
  - 结果会包含候选项、术语、关系和待人工复核内容的统计。

### Master Update

- `Merge Masters`
  - 把更新文件夹里的多个主表类文件合并进当前主表。
  - 适合合并多个来源的主表增量内容。
  - 支持拖拽调整处理顺序，最上面的文件先处理。

- `Source Text`
  - 用更新文件夹里的来源文本，更新主表中的源文。
  - 适合主表源文需要整体刷新或补充新 key 时使用。
  - 支持拖拽调整处理顺序，最上面的文件先处理。

- `Translation`
  - 用更新文件夹里的译文，更新主表中已有 key 的翻译内容。
  - 更适合在主表结构已经稳定后，补充或覆盖译文。
  - 支持拖拽调整处理顺序，最上面的文件先处理。

- `Source+Translation`
  - 一次完成两段更新：
    - 先跑 `Source Text`
    - 再跑 `Translation`
  - 需要分别选择源文更新文件夹和译文更新文件夹。
  - 两段处理顺序固定，但每一段内部文件顺序可以拖拽调整。

## 最短操作步骤

1. 先备份要处理的 Excel 文件。
2. 关闭 Excel 里已经打开的相关文件，避免文件被锁定。
3. 打开程序后，进入对应页签。
4. 按界面提示选择主表、来源文件、目标文件夹、规则文件或输出文件。
5. 检查列号是否正确，再点击当前页签里的运行按钮。
6. 等待底部状态栏显示 `Done:`。
7. 如果出现 `Failed:`，点击右下角 `View Logs` 查看原因。

## 常见提醒

- 最稳定的运行环境是 Windows。
- 部分功能依赖本机 Microsoft Excel 和 `pywin32`，尤其是 `Compatibility`。
- 程序一次只会执行一个任务；任务未结束时，其他按钮会暂时不可用。
- 涉及覆盖、删除、批处理前，先用一小份样例文件试跑一遍更安全。
- `Column Clear -> 删除列`、`Deep Replace`、批量覆盖类操作都建议先备份。
